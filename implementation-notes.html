Composer and Statusline Decisions

  • The existing TUI footer source has model, reasoning effort, and cwd, but no explicit mode. The implementation keeps the current hard-coded mode as code.
  • CTX is derived from mimir::model_catalog::ctx_label. Live TUI refresh builds a provider-qualified lookup key for the catalog, while the rendered MODEL value remains the existing bare model id to avoid making the statusline noisier.
  • Reasoning effort is rendered as the optional model flag token after MODEL instead of as a separate EFFORT field. This matches the spec's allowance for compact model option flags and avoids adding a fifth statusline field.
  • Branch remains parsed from the existing cwd string for compatibility, but it is lower priority than MODE, MODEL, CTX, and CWD; on narrow widths it is the first field omitted.
  • When no footer/model selection has been sourced yet, Iris continues to omit the statusline rather than inventing unknown model/context/cwd values.
  • On terminals too short to fit the canonical composer shape, the hint row is hidden rather than overlapping the editor text. This preserves width/row integrity under severe height constraints.
  • The bare/qualified model adapter on Screen::set_footer is test-only; production uses set_footer_with_context so release builds do not carry an unused compatibility shim.
  • The statusline dot aligns with the editor input text by using the existing composer border indent plus the two-column editor text padding. A broader shared-gutter change was avoided because it reflows transcript and pasted-terminal rendering outside this feature's scope.
  • The canonical editor vertical chrome count is named in code to keep the height calculation and text-area subtraction in sync.

Working Indicator Decisions

  • Telemetry source: the TUI only receives provider token usage on ProviderTurnCompleted. The indicator therefore shows usage once the active turn has reported it, and clears stale prior usage when a new turn starts so the readout represents the current turn rather than the previous one.
  • Field scope: the prior active-turn line included model reasoning effort text. The new indicator intentionally omits that extra field because the spec fixes the working indicator field order to activity, elapsed, interrupt, then token telemetry.
  • Merge with composer statusline: after merging origin/main, the working indicator remains outside editor chrome and is appended to the transcript tail before composer/statusline chrome. This preserves the composer statusline PR's height model while matching the working indicator's transcript-flow placement.

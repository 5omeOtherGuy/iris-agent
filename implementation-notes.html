Part A — markdown

  • Theme as patch-composed Styles (not pi-mono's style functions). MarkdownTheme holds ratatui Style fields plus a base style that is patch-merged under every emitted span (the Rust analogue of pi-mono's defaultTextStyle). Emphasis stacks by patching (heading/bold/italic/ strikethrough/quote). MarkdownTheme::default() is constructed so the patched result equals the previous hardcoded dim()/fg(Cyan) output exactly, keeping every existing snapshot byte-identical. Verified by the full pre-existing test set plus default_theme_matches_legacy_inline_code_color.
  • Highlight hook is Option<Rc<dyn Fn(&str, Option<&str>) -> Vec<Line>>> (no bundled highlighter). To call it I had to buffer the fenced block (capture lang at Start(CodeBlock), accumulate text, render at End) instead of the old per-Text-event emission. Default (no hook) path preserved; one behavioral nicety: internal blank lines inside code are now kept (the old per-chunk logic dropped them); only the trailing-newline empty segment is dropped. No existing test covered internal blanks.
  • Link styling seam. theme.link is patched onto link-label spans; default is a no-op Style so output is unchanged, and the seam is proven by link_theme_seam_styles_label. The visible destination is still appended. Raw OSC-8 emission is deferred (see below).
  • Tables need a width but the renderer is width-agnostic by design. Iris stores pre-rendered rows and re-wraps them per frame; markdown runs once at append time. Tables are the one block that cannot be re-wrapped line-by-line without corrupting the box, so I threaded a table-layout width into render_markdown_themed. pane.rs passes content_width - ASSISTANT_TEXT_PREFIX so a full-width table line plus the "\u25cf "/" " marker prefix still fits the frame and the downstream word-wrapper is a no-op on table lines. transcript.rs uses last_width (fallback DEFAULT_RENDER_WIDTH=80 before the first frame).
  • Table layout. Column widths = natural cell widths when they fit width - (3*cols+1) border overhead, else proportional shrink with a floor of 1 and rounding-drift fixup (fit_columns). Cells wrap (wrap_plain, word-wrap + hard-break of over-long words) so every physical line fits; row height = tallest cell. Alignment from pulldown_cmark::Alignment. If width is too narrow for a stable box, it falls back to pipe-joined plain rows (narrow_table_falls_back_to_plain_rows). Cells collect plain text (inline emphasis inside cells not preserved) to keep width math and wrapping simple — a deliberate scope cut.
  • Known limitation (documented, not fixed): because table layout is computed once at append time, a table does not reflow on terminal resize; narrowing can let the generic wrapper break a previously-fitting box. This matches the existing retained-panel behavior. Non-table markdown reflows normally (it is word-wrapped at render time).
  • render_markdown(text) (default theme + default width) is now used only by tests, so it is #[cfg(test)] to keep clippy -D warnings clean; production callers use render_markdown_themed.

Part B — reasoning / thinking blocks

  • New event is a DISPLAY event, block-level. AgentEvent::AssistantReasoning { text, redacted } sits with AssistantText (which already carries model text), not the ADR-0020 observability family — so carrying reasoning text is consistent. It is block-level (emitted once per ReasoningBlock) because reasoning is assembled into whole blocks at turn completion; the provider stream exposes no reasoning deltas on this path. See ADR-0024.
  • Storage stays intact (ADR-0016). Emission was added before the unchanged Message::assistant_reasoning_block(block) push at nexus.rs; the row, its continuity, redacted flag, and ModelOrigin are written exactly as before. Test completed_turn_records_reasoning_and_all_tool_calls_before_results (pre-existing) still passes; new tests assert the row content is preserved alongside emission.
  • Redacted handling. A redacted block is emitted with empty text + redacted: true — the opaque content is still stored for replay but never leaves Nexus as text and is never rendered (redacted_reasoning_emits_event_without_leaking_text, redacted_reasoning_never_renders_trace_text). The TUI shows a "[reasoning withheld by provider]" placeholder.
  • Thinking block reuses the existing panel + ctrl+o toggle. Rendered as a panel titled THINKING with a Thinking... right-label, expanded:false by default. The existing collapse render-loop hides the body until the existing toggle_latest_panel (ctrl+o) expands it — no new affordance. Default-collapsed mirrors pi-mono's hideThinkingBlock default and keeps thinking quiet by default. The trace is markdown-rendered with MarkdownTheme::thinking() (dim+italic base). Reasoning events are emitted before the assistant-text event, so the block renders above the answer (reasoning_renders_before_assistant_text).
  • Stop reasons (B8) were already implemented and are preserved, not re-built. The requested surfacing — reuse completion_reason_notice — already exists at nexus.rs: MaxOutputTokens / ContextWindowExceeded / content-less Refusal emit a Notice after the assistant content (tested), aborted via the interrupt notice, error via the turn-error path. My changes keep this ordering (reasoning -> text -> completion notice). Adding a redundant stop-reason channel would be over-engineering, so none was added.
  • Text fallback (ui/text.rs) ignores reasoning by design (different surface, out of scope); the arm is explicit so the exhaustive match compiles and the fallback's output is unchanged.

Deferrals (named) and what unblocks them

  • Raw OSC-8 hyperlink emission — deferred. Iris's terminal surface strips control sequences and has no OSC-8 passthrough; that is the text-engine task. This change provides link styling + the theme.link seam only. Unblocked when the text engine adds OSC-8 passthrough.
  • Bundled syntax highlighting — deferred (a dependency decision). The highlight_code hook ships with no built-in implementation; default fenced blocks stay dimmed. Unblocked by a decision to add a highlighter crate that wires a closure into the theme.
  • Images / terminal graphics; the Component / theme object model; markdown in the text fallback — out of scope (separate tasks/surfaces).
  • Streaming reasoning deltas — not applicable here; reasoning arrives block-level. Unblocked if a provider/stream surfaces incremental reasoning.
  • Table reflow on resize — see Part A limitation; unblocked by re-running markdown on resize or storing table source for re-layout.

Wave-1 review fixes (commit 2)

  • Streaming + reasoning ordering/duplication (real bug, fixed). The reasoning apply-arm called finish_stream() (and push_thinking_block used begin_block), which committed the live-streamed answer before the thinking block and then double-committed it when AssistantTextEnd re-committed the same text. Fix: the reasoning arm no longer finishes the stream; push_thinking_block uses push_blank so the panel is inserted above the still-pending stream, which commits once afterwards. Test: reasoning_renders_above_streamed_answer_without_duplication.
  • Streaming table width (real bug, fixed). render_streaming_assistant is handed the full frame width but passed it straight to table layout; committed rows get the already-inset content width. Fix: streaming reduces by 2*TEXT_COLUMN_X_PADDING first so both paths lay out tables at full-10. Test: assistant_table_never_exceeds_frame_width (widths 40/60/80, both paths).
  • Verified false positives (no change): several reviewers flagged the committed assistant table width and the thinking-block panel width as too wide. Checked the constants: last_width = full - 2*TEXT_COLUMN_X_PADDING = full-8 and panel body width = panel_width(full) - PANEL_BODY_CHROME_WIDTH = (full-4) - 4 = full-8 are equal, so both already match. No change made (avoided an incorrect double-subtraction).
  • Hardening: table cell lines are clamped with a width-aware truncate_to_width so an unsplittable wide glyph in a 1-column slot can't push the border out; HardBreak inside a cell becomes a space; table overhead/column-sum use saturating arithmetic.

Wave-2 review fixes (commit 3)

  • Nested-table prefix (real bug, fixed). A GFM table inside a blockquote or list rendered at column 0, dropping the > markers / list indent (confirmed empirically). render_table now computes the block prefix (quote markers + list-continuation spaces), reduces the table layout width by the prefix width, and prepends the styled prefix to every emitted table line. Top-level tables (prefix empty) are unchanged. Tests: blockquoted_table_keeps_quote_prefix, nested_list_table_keeps_indent_and_fits_width.
  • Narrow-table fallback now wraps (fixed). The pipe-joined fallback previously ignored width; it now wraps each fallback row to the available width so even the fallback never overflows.
  • Verified false positive (no change): thinking-block panel width. Two reviewers claimed the panel body is W-12 vs the W-8 used. Checked the code: panel_body_lines uses panel_width(W) - PANEL_BODY_CHROME_WIDTH = (W-4) - 4 = W-8, exactly equal to markdown_content_width() (last_width = W-8). They match; tables in thinking blocks fit. No change.
  • Pre-existing / out of scope (not changed): panel_end_from terminate-on-missing-Bottom (my panels are always well-formed with a Bottom in one synchronous push, so it cannot loop), the render-loop collapsed-reset for hypothetical nested panels (correct for today's linear panels), ProviderTurnError being display-ignored (unchanged from before), and the streaming markdown re-parse per frame (already per-frame before this work). Inline code ignoring surrounding emphasis and the list-marker-on-first-line behavior are unchanged from the original renderer (the latter is handled by continuation_prefix).
  • Micro-optimizations declined (per the smallest-correct-change rule): per-row String allocations in table wrap/truncate, MarkdownTheme::default() construction cost, and the single per-code-block Rc clone are negligible relative to the bounded, truncated streaming preview; caching streamed markdown lines is a larger pre-existing optimization left for separate work.
